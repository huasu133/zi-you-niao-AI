# 自由鸟 v4 — 高级项目经理审查报告

> 审查人：SeniorProjectManager | 审查日期：2026-05-29

---

## 总体评估

**这是一份优秀的技术蓝图，但作为项目计划缺少三个关键要素：验收标准、风险预案、用户指南。**

### 优势
- 工具拆分清晰，每个文件职责单一
- 安全防护全面（SSRF/路径遍历/命令注入/脱敏）
- 搜索方案灵活（DDG 主搜 + Tavily/ContextWire 备选）
- 记忆系统设计合理（长期记忆 + 每日日志）

### 欠缺
- **没有验收测试** — 如何知道"搭好了"？
- **没有调试指南** — 出错了怎么办？
- **没有"Done"的定义** — 什么算完成？

---

## 关键问题

### 1. 工期严重低估

文档写"工期：半天"（4小时），按实际代码量评估：

| 阶段 | 乐观估计 | 现实估计 |
|------|---------|---------|
| 建项目 + 安装依赖 | 10min | 20min（含 npm 安装）|
| 写 7 个工具文件 | 60min | 90min（含调试）|
| 写 mcp-client.js | 20min | 30min（SDK 适配）|
| 写 tool-registry.js | 30min | 60min（FC schema 调试）|
| 写 app.js | 40min | 60min（SSE 流调试）|
| 写 expert-router.js | 20min | 40min |
| 写 Web UI | 40min | 60min（CSS 调整）|
| 写 soul.md + 专家定义 | 20min | 30min |
| 修 bug + 调试 | 0min | 45min |
| **总计** | **~4h** | **~7h（一个整天）** |

**建议：工期改为「一个工作日」，半天是纯写代码的时间，调试和修 bug 至少翻倍。**

### 2. 缺少启动验收清单

文档没有告诉用户"怎么知道搭成功了"。建议加一个验收步骤：

```
✅ 启动验收清单
□ node app.js 启动无报错
□ 浏览器访问 http://localhost:3456 看到 Web UI
□ 发送"你好"得到 AI 回复
□ 发送"搜一下 Node.js 最新版本"触发搜索并返回结果
□ 发送"读一下 package.json"触发 read_file 工具
```

### 3. 没有错误排查指南

如果出问题了，用户没有任何指导。建议加一个 FAQ 节：

```
⚠️ 常见问题
Q: 启动报 MODULE_NOT_FOUND
A: 确认 `npm install` 执行成功，检查所有 require 路径

Q: 搜索返回无结果
A: 如果不配 API Key，DuckDuckGo 可能被限流，等待几分钟重试

Q: DeepSeek API 返回超时
A: 检查 DEEPSEEK_API_KEY 是否正确，网络是否能访问 api.deepseek.com
```

### 4. 功能范围建议（MVP vs 全部）

当前方案一次实现了所有功能，但有些功能是"想要的"而不是"必需的"：

| 功能 | 优先级 | 理由 |
|------|:------:|------|
| 7 个核心工具 | 🔴 P0 | 核心功能 |
| Web UI 聊天 | 🔴 P0 | 没有就不能用 |
| 搜索（DDG） | 🔴 P0 | 核心功能 |
| 搜索（Tavily） | 🟡 P2 | 锦上添花，DDG 兜底 |
| 搜索（ContextWire） | 🟢 P3 | 注册都关了，别等了 |
| 专家路由 | 🟡 P2 | 高级功能，非核心 |
| GitHub 连接器 | 🟢 P3 | 你已经有终端了 |
| 浏览器操控 | 🟢 P3 | 可选依赖，可后装 |
| 任务管理 | 🟡 P2 | 有用但非必需 |

**MVP 建议：先跑通核心 7 工具 + Web UI + DDG 搜索，专家/Tavily/GitHub 浏览器后加。**

## 风险登记册

| 风险 | 概率 | 影响 | 缓解措施 |

| DeepSeek API 响应慢 | 中 | 中 | 加超时处理，显示"思考中"状态 |

| 搜索查询脱敏误拦正常内容 | 低 | 低 | 用户反馈后调整正则 |
| 记忆文件无限增长 | 高 | 低 | 需要增加归档策略（当前未实现） |
| 多人同时使用（本地不可能） | 低 | 高 | 本地单人使用，无此风险 |

## 结论

**建议：按 Phase 1 → Phase 2 → Phase 3 分阶段实施，MVP（Phase 1）约一个下午完成。** 

建议先跑通核心，别一口气把所有功能全搭上。等 Phase 1 跑通了再决定要不要加专家路由和浏览器操控。
